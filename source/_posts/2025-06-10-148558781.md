## CD/DVD 驱动器挂载

### 物理层面连接 ISO

不管是使用虚拟机还是实体机安装都可以使用 CD/DVD 驱动器挂载。

以虚拟化平台为例：

![](/images/2c51a477770869256376d81c99ac0b76.png)

![](/images/edba7c6af0823cdfa5e24106497f24ce.png)

按上图配置完成后即可。

### 主机层面挂载 ISO

物理层面先连接 ISO 之后，还需要在主机层面挂载 ISO 才算完成。

主机层面挂载 ISO 的命令可以有以下方式，在 root 用户下执行：

```
[root@ ~]# mount /dev/sr0 /mnt
## 或
[root@ ~]# mount /dev/cdrom /mnt
```

以上两条命令的效果一样，都可以。如果没有在物理层面连接 ISO，主机层面挂载 ISO 时会报错：

![](/images/59180a304542d6c15ea11c452e405201.png)

遇到这个报错就需要在物理层面先连接 ISO，然后再次执行：

![](/images/f06efe93f007350694bf1b8394796841.png)

执行 `df -h | grep /mnt` 查看是否挂载成功：

![](/images/3dfbfb49ecea5781cf674cd02596f32d.png)

如果查询结果如上，则代表已经挂载成功了。

## 手工上传镜像挂载

### 物理层面上传 ISO

这个就很简单了，上传下载好的安装镜像 iso 到服务器主机指定目录下，以 `/iso` 为例：

![](/images/9236cbf284a2afab894787104301095c.png)

上传之后就可以，烦在烦在 ISO 比较大，下载上传比较慢。

### 主机层面挂载 ISO

主机层面挂载 ISO 的命令，在 root 用户下执行：

```
mount -o loop /iso/OracleLinux-R9-U6-x86_64-dvd.iso /mnt/
```

![](/images/c1ad15843941452ccbae5d63b12cb548.png)

执行 `df -h | grep /mnt` 查看是否挂载成功：

![](/images/c4ed6aaa0ce4d98b625a50ea295ea58e.png)

如果查询结果如上，则代表已经挂载成功了。
