Oracle 数据库通常建议
开启
归档模式，记录数据库操作的记录归档到本地日志文件！

可以通过以下
命令
查看是否开启归档：

```
archive log list


AI写代码sql



* 1
```

如图，即已开启归档！

**有啥好处呢？**

开了归档之后，可以 
RMAN
 实时备份数据，配置 DataGuard 必须开启归档模式，数据库基本能保证零丢失！

**缺点可能就是：**

1、占用大量磁盘空间  
2、占用部分
系统
资源  
3、如果归档空间满了，可能导致数据库宕机

**总的来说，利大于弊，现在谁还差那么点磁盘空间和系统资源呢？对吧！**

**开启归档模式的步骤：**

**1、设置归档路径**

```
alter system set log_archive_dest_1='LOCATION=/archivelog';


AI写代码sql



* 1
```

这个 `/archivelog` 目录需要是磁盘中物理存在的路径，否则无法归档！  
![](/images/4ed2bdb734d312780d78d761f66ee34d.png)  
**2、关闭数据库，开启到mount状态**

```
shutdown immediate
startup mount

```
alter database archivelog;
